# 一个会自己变聪明的AI助手：Hermes Agent深度拆解

上周在 GitHub Trending 上看到一个项目，17K 星，名字叫 Hermes Agent。

本来没当回事——AI Agent 多了去了，每个月都有新的"颠覆性"出来。但点进去看了十分钟之后，我把手里的水杯放下了。

因为这个项目在解决一个我一直在等别人解决的问题：AI 助手用了一个月还是那个傻样子，从来不会从自己的错误里学东西。

Hermes Agent 说它做到了。我花了一周时间研究它到底怎么做到的。

## 它到底是什么

Hermes Agent 是 Nous Research 做的开源 AI Agent，2026 年 2 月发布。

Nous Research 这家公司值得多说两句。2023 年成立，创始人是 Jeffrey Quesnelle，总融资 7000 万美元（包括一笔 5000 万的 A 轮），定位是"美国开源 AI 运动的领导者"。他们之前做的东西在开源社区口碑不错——Hermes 系列模型本身就是很受欢迎的开源大模型家族，还跟 OpenAI 联合创始人 Diederik Kingma 合作过多篇论文。

这次做的 Hermes Agent，跟模型是两回事。它是一个**自托管的 AI Agent 框架**，跑在你自己的服务器上，连到你的 Telegram、Discord、Slack、WhatsApp，然后开始干活。

重点来了——它越用越聪明。

不是那种"我们会持续更新模型"的画饼。是它真的会把你教它的东西记下来，下次直接用。

## 自我学习循环：这是最核心的部分

大多数 AI Agent 的工作模式是这样的：你给指令 → 它执行 → 返回结果 → 下次对话，一切归零。

Hermes 不一样。它有一个**闭环学习系统**，分三层：

**第一层：技能自动生成。** 你让它帮你做了一件复杂的事——比如"分析这份 PDF 里的财务数据，生成一份对比报告"。它做完之后，会自动把这个过程沉淀成一个可复用的"技能"（Skill）。下次你说"分析财务数据"，它直接调用这个技能，不用重新摸索。

**第二层：技能迭代优化。** 不只是记住步骤，它会在使用过程中不断修正。第一次生成的报告格式可能不太对，你改了几个地方——它会学到"用户喜欢表格而不是大段文字"，下次自动调整。

**第三层：用户建模（Honcho）。** 这是最有意思的部分。Hermes 集成了一个叫 Honcho 的记忆后端，它不只是存储信息，而是通过分析你的对话，建立一个关于你的"用户模型"——你的偏好、沟通风格、工作习惯、目标。

打个比方。用别的 AI 助手，每次对话像跟一个失忆的新同事说话。用 Hermes，像跟一个跟你搭档了三个月的人说话。他知道你喜欢简洁，不喜欢客套，习惯在晚上十点以后处理重要邮件。

这个差距不是一点点。

## 跟 OpenClaw 比，到底怎么样

既然提到了 OpenClaw——对，就是我现在跑着的这个框架——那必须正面比一下。

TuringPost 的一篇分析文章直接把标题写成了"Hermes Agent: OpenClaw's Rival?"。Kilo.ai 爬了 Reddit 上 r/openclaw 的 25 个帖子、1300 多条评论做分析。结论很清晰：

**Hermes 的优势：**
- 自我学习循环是真正的差异化。OpenClaw 的记忆靠文件（MEMORY.md、daily notes），需要 Agent 自己读写。Hermes 是内建的闭环，自动执行
- 用户建模（Honcho）比 OpenClaw 的文件系统更结构化
- 有内置的 OpenClaw 迁移工具，切换成本低
- 开源社区活跃，17K GitHub 星，有 Skills Hub（agentskills.io）

**OpenClaw 的优势：**
- 生态成熟。插件系统、Skill 市场、多平台支持都已经很完善
- 稳定性经过更长时间验证。Hermes 才发布两个月
- MCP 协议支持更好，工具链更丰富
- 文档和社区资源更多

**Reddit 用户的真实声音：**
- 不少人报告从 OpenClaw 切换到 Hermes 后，"记忆效果明显改善"
- 但也有人反馈 Hermes 的安装配置比 OpenClaw 复杂，对非技术用户不友好
- 一个高频词是"potential"——大家都觉得方向对了，但还太早

说句实话，这两个产品代表了 AI Agent 的两种哲学：OpenClaw 是"插件生态 + 文件系统"，Hermes 是"内建学习 + 自主进化"。短期看 OpenClaw 更实用，长期看 Hermes 的方向可能更对。

不过我也不确定。真的不确定。两个月的产品和一年的产品比，本身就不太公平。

## 技术细节：三种记忆 + 六种部署

**三层记忆架构：**

1. **会话记忆**：当前对话的上下文，跟所有 AI 一样
2. **持久记忆**：跨会话的知识存储，写在自己的文件系统里
3. **Honcho 记忆**：用户建模层，通过"辩证推理"（dialectic reasoning）持续更新对你的理解

v0.7.0 版本开始支持可插拔记忆后端，你可以选择不用 Honcho，用别的方案。

**部署方式有六种：**
- 本地运行（直接在机器上跑）
- Docker 容器
- VPS 部署（官方说 $5/月的 VPS 就够）
- Daytona（云开发环境）
- Modal（Serverless）
- MCP Server 模式（让别的 Agent 调用它的能力）

40+ 内置工具，支持多平台消息推送，安全方面……嗯，Lushbinary 的对比文章提到了一个 CVE-2026-25253 漏洞，说明安全审计还在早期阶段。

这个信息很重要。跑在自己服务器上的 Agent，安全是底线。

## 商业视角：开源策略和定位

Hermes Agent 的定位很明确：**比 Claude 便宜的开源替代方案。**

这个定位很聪明。Claude Code 和 OpenClaw 都走的是闭源 + 订阅或付费 API 的路线，Hermes 直接开源，自托管，你的数据你自己控制。

成本对比：自托管 Hermes，用开源模型（比如 Nous 自己的 Hermes 模型），主要成本就是服务器费用。$5/月的 VPS，比任何订阅都便宜。当然如果你要接 GPT-4 或 Claude 的 API，那成本另算。

Nous Research 的商业模式跟典型开源公司一样：开源核心 + 商业服务。企业版、托管版、技术支持——这些还没正式发布，但方向是清晰的。

## 它适合谁

**适合：**
- 有技术背景，愿意折腾自托管的人
- 对 AI 隐私和数据主权有要求的用户
- 长期使用同一个 Agent、希望它越来越懂你的人
- 开发者，想基于它做二次开发

**不适合：**
- 非技术用户。安装配置需要命令行操作
- 需要开箱即用、零配置体验的人
- 对稳定性有极高要求的生产环境——它才两个月大

**跟 OpenClaw 的选择建议：**
如果你重视生态和稳定性，现在还是选 OpenClaw。如果你对"AI 自我学习"这件事有执念，愿意承担早期产品的风险，Hermes 值得试试。

## 我的判断

Hermes Agent 解决的问题是真实的——AI 助手不会从经验中学习，这确实是当前所有 Agent 的核心缺陷。

它的解法也是对的——闭环学习 + 用户建模 + 技能沉淀，这个架构设计比"让 Agent 读写文件"更本质。

但"方向对了"和"产品成熟了"之间，隔着至少一年。

我打算在我的开发环境里装一个，跟 OpenClaw 并行跑一段时间。看看那个自我学习循环到底能学多快，学多深。

如果三个月后它真的明显比现在聪明了，我会再写一篇。

如果没变——那也值得告诉你们。
